# PolySeek タグ付けガイドライン

## 概要

PolySeekは、VRChat向けBOOTH商品をタグベースで検索できるWebアプリケーションです。このガイドラインは、商品に適切なタグを付与し、ユーザーが効率的に商品を検索できるようにすることを目的としています。

タグ付けシステムは、Danbooru等のBooru系サイトのタグシステムを参考にしており、以下の特徴があります:

- **タグカテゴリ**: タグは複数のカテゴリに分類され、カテゴリごとに色分けされます
- **レーティングシステム**: 商品の内容に応じて4段階のレーティングを付与します
- **タグ含意**: あるタグが付けられると、自動的に関連タグが追加されます
- **タグ翻訳**: 日本語と英語のタグが相互に翻訳されます
- **タグ階層**: 親子関係を持つタグにより、より詳細な分類が可能です

## ガイドライン構成

### 1. レーティングガイドライン

商品の年齢制限や内容の適切性を判断するためのガイドラインです。

- [レーティングガイドライン詳細](./rating-guidelines.md)
- [レーティング判定フローチャート](./rating-flowchart.md)

**レーティングカテゴリ**:
- **General (全年齢)**: 誰でも安全に閲覧できる内容
- **Sensitive (軽微な配慮が必要)**: わずかに刺激的、または示唆的な内容
- **Questionable (要注意)**: 性的または暴力的な要素を含むが、露骨ではない内容
- **Explicit (成人向け)**: 露骨な性的表現や極端な暴力表現を含む内容

### 2. タグカテゴリガイドライン

タグは以下のカテゴリに分類されます:

- **Rating (レーティング)**: 年齢制限・内容区分
- **Character (キャラクター)**: VRChatアバター、キャラクター種類
- **Clothing (衣服)**: 服装、アクセサリー
- **Body (身体)**: 体型、身体的特徴
- **Scene (シーン)**: 背景、環境、シチュエーション
- **Style (スタイル)**: アートスタイル、技法
- **Meta (メタ)**: 商品タイプ、ファイル形式、対応プラットフォーム
- **General (一般)**: その他のタグ

詳細は[タグカテゴリ一覧](./tag-categories.md)を参照してください。

### 3. タグ付けのベストプラクティス

#### タグの命名規則

- タグ名は小文字で統一する(システムが自動的に正規化)
- 日本語タグと英語タグを両方用意する
- スペースの代わりにアンダースコア(_)を使用する
- 略語よりも完全な単語を使用する

#### タグの付け方

1. **具体的に**: できるだけ具体的なタグを使用する
   - 良い例: `vrchat_avatar`, `fullbody_model`
   - 悪い例: `model`, `asset`

2. **適切な数**: 商品ごとに5〜20個程度のタグが推奨
   - 少なすぎると検索で見つかりにくい
   - 多すぎるとノイズが増える

3. **レーティングタグは必須**: すべての商品に1つのレーティングタグを付与

4. **公式タグを優先**: BOOTH由来のタグがある場合は優先的に使用

5. **含意を活用**: 親タグが自動的に追加されるため、詳細なタグのみを手動で追加

#### 避けるべきタグ

- 主観的すぎるタグ: `beautiful`, `cool`, `awesome`
- 曖昧なタグ: `other`, `misc`, `various`
- 重複タグ: 既存タグと同じ意味のタグ
- 個人情報: 作者の個人情報を含むタグ

### 4. タグ検索の方法

PolySeekでは、以下の検索機能を利用できます:

- **AND検索**: `tag1 tag2` (両方のタグを含む商品を検索)
- **OR検索**: `tag1 OR tag2` (いずれかのタグを含む商品を検索)
- **NOT検索**: `-tag` (特定のタグを除外)
- **ワイルドカード**: `tag*` (前方一致検索)
- **レーティング検索**: `rating:general`, `rating:explicit`

## 実装状況

### 現在実装済み

- タグモデル(Tag)
- タグカテゴリモデル(TagCategory)
- 商品タグ関連(ProductTag)
- タグ翻訳機能(TagTranslation)
- タグ含意機能(TagImplication)
- タグ階層機能(TagHierarchy)
- タグ編集履歴(TagEditHistory)
- ユーザーのセーフサーチ設定(isSafeSearchEnabled)
