## 2. 論文構成

### 2.1 セクション構成

```
1. 要旨
2. 背景
3. 目的
4. システム設計
5. 実装
6. 考察
7. 結論
```

### 2.2 各セクションの内容

#### 1. 要旨（200-300字程度）
研究の概要、目的、手法、成果を簡潔にまとめる。

#### 2. 背景
VRChatとBOOTHの現状、検索における課題を述べる。

**含めるべきデータ**:
- 2024年度 3Dモデルカテゴリ注文件数: 約402万件（前年比101%増）
- 2024年度 取引高: 58億円（前年比187%増）

**課題の記述**:
- BOOTHのタグは出品者のみ編集可能
- タグ付けがまばらで、活用が限定的
- ファッション用語やジャンルタグの定義が曖昧
- キーワード検索中心で目的の商品が見つけにくい
- 人気順の基準が不明瞭

**注意**: Booru系サイト（Danbooru等）には言及しない。BOOTH問題に焦点を当てる。

#### 3. 目的
PolySeekが解決する問題と達成目標を明確に述べる。

**コアコンセプト**:
- 誰でもタグを付与・削除できるシステムの構築
- ユーザーコミュニティ主導のデータベース作成
- 検索体験の向上

#### 4. システム設計
アーキテクチャとタグシステムの設計を両方バランスよく記述する。

**アーキテクチャ**:
- Next.js (App Router) によるWebアプリケーション
- Prisma ORM + PostgreSQL（Docker環境）
- Google OAuth認証

**タグシステム設計**:
- タグ含意（implication）
- タグ階層（hierarchy）
- タグエイリアス（alias）
- タグ翻訳

#### 5. 実装
技術スタックと主要機能の実装詳細を記述する。

**技術スタック**:
- フロントエンド: Next.js (App Router), React, Tailwind CSS
- バックエンド: Next.js API Routes
- データベース: PostgreSQL（Docker）
- ORM: Prisma
- 認証: NextAuth.js + Google OAuth
- 本番環境: OCI（Nginx + PM2）

**主要機能**:
- 商品検索・フィルタリング（マイナス検索対応）
- タグ管理システム
- レーティング・セーフサーチ
- 商品登録（BOOTHからのスクレイピング）

#### 6. 考察
実績データと今後の課題を述べる。

**実績データ**:
- 利用人数: 9人
- 登録タグ: 92個

**今後の課題**:
- スケーラビリティ
- コミュニティ拡大
- タグ品質の維持

#### 7. 結論
成果のまとめと今後の展望を述べる。

## 4. 既存背景テキスト（参考）

以下は過去のプレゼン資料からの引用で、背景セクションの元データとして使用する:

> VRChat内で使用できるアバターやアセットの多くは、日本のクリエイター向けマーケットプレイスであるBOOTH.pmで取引されています。VRChatのユーザー増加に合わせてBOOTHの利用ユーザーも増加し、2024年度の3Dモデルカテゴリの注文件数は約402万件を超えています。これは前年比で101%の注文件数の増加です。取引高に関しても2024年度は58億円、前年比187%増という記録的な増加が見られています。
>
> BOOTHはVRChatと提携しており、VRChatカテゴリのアバターだけでも多数の商品がこのように出品されています。BOOTHを利用して商品を購入するユーザーが増えるにつれ、BOOTHへVRChat向けアイテムを出品するクリエイターの数も増えています。
>
> BOOTHの取引高が増える一方で、3Dモデルカテゴリの商品数増加に伴い、ユーザーが「欲しい」と思う商品を効率的に見つけ出すことが難しくなってきています。現在、BOOTHでは商品に含まれる要素に関するタグをつけることができますが、このタグは出品者のみが編集可能であり、出品者によって商品につけるタグの個数はまばらで、デザインに関するファッション用語のタグの活用が限定的であったり、サイバーパンクやファンタジーといった商品が属するジャンルのタグの定義があいまいな状態です。
>
> 現在のBOOTHの検索機能はキーワード検索が中心であり、膨大な商品の中から目的のものを探し出すには多くの時間と労力がかかっています。この点が、VRChatユーザーが抱える大きな課題の一つであると認識しています。
>
> 例として、この画面では検索欄に「VRChat」と「こたつ」というキーワードを入力しているのですが、検索結果にはこたつそのもののアセットもあれば説明文にこたつが含まれている衣装やアセットも表示されてしまっています。さらに、BOOTHの問題点としてすでに人気の商品だけが注目されやすいという問題があります。この画像では商品の並び順は初期状態で人気順になっています。この人気順は基準が不明瞭であり、こたつを探しているのに衣装が表示されるといった適切ではない検索結果が表示される一因になっています。
>
> 先ほど述べたBOOTHのタグの問題点である「タグの活用が限定的」という点や、「適切ではない検索結果」といった問題に対し、本制作では、BOOTHの商品に対して、アプリケーション内で誰でもタグを付与・削除できるシステムを構築し、ユーザーコミュニティ主導のデータベースを作成を行うことで検索体験を向上させることが問題の解決方法だと考えています。
