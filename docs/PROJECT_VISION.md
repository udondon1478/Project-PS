# PolySeek プロジェクトビジョン

**作成日**: 2026-01-18
**目的**: プロジェクトの方向性を明確にし、開発時の判断基準とする

---

## 1. プロジェクトの本質

### 一言で表すと

> **VRChat向けBOOTH商品のための、高品質なタグベース検索エンジン**

### コアコンセプト

- **Booru系サイト（Danbooru等）のタグ検索システム**を**VRChatアセット検索**に応用
- ユーザーが「本当に欲しいアイテム」に効率的に辿り着けるようにする
- **検索特化型**のシステム（ソーシャル機能は最小限）

### 提供価値

| 既存の問題 | PolySeekの解決策 |
|------------|------------------|
| BOOTHでは「対応アバター」「色」「スタイル」等での絞り込みが困難 | 詳細なタグによる高度な検索 |
| 商品の発見性が低い | タグ含意・階層を活用した網羅的な検索結果 |
| メタデータの不足 | コミュニティ主導のタグ付けによるデータ充実 |

## 3. 機能方針

### 方向性: 検索特化型システム

**目的**: 高品質なデータベース構築と最高の検索体験の提供

```
┌─────────────────────────────────────────────────────────┐
│                    PolySeekの重点領域                    │
├─────────────────────────────────────────────────────────┤
│                                                         │
│   検索品質        タグシステム        データ収集         │
│   ★★★★★         ★★★★★           ★★★★★          │
│                                                         │
│   整理機能        評価機能          ソーシャル          │
│   ★★★☆☆         ★★☆☆☆           ★☆☆☆☆          │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### 機能実装方針

#### 実装する機能

| 機能 | 優先度 | 詳細 |
|------|--------|------|
| タグ検索・フィルタリング | 最優先 | マイナス検索、含意、階層を含む高度な検索 |
| タグシステム | 最優先 | 含意、階層、エイリアス、翻訳 |
| 星評価システム | 中 | 5段階の星評価（文章レビューなし） |
| コレクション/プール | 中 | 商品を整理するテーマ別コレクション |
| レーティング・セーフサーチ | 高 | 4段階レーティングによるフィルタリング |

#### 実装しない/慎重に判断する機能

| 機能 | 方針 | 理由 |
|------|------|------|
| フォロー機能 | **不要** | 検索特化の方針と合わない |
| DM機能 | **不要** | ソーシャル機能は最小限に |
| コメント機能 | **慎重に判断** | 必要性を見極めてから検討 |
| 文章レビュー | **不要** | 星評価のみで十分 |

## 4. ユーザー権限モデル

### 権限レベル

```
┌─────────────────────────────────────────────────────────┐
│                      権限階層                            │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  管理者 (ADMIN)                                         │
│  ├── 全ての操作が可能                                    │
│  ├── タグエイリアス・含意・階層の承認                      │
│  ├── ユーザー管理                                        │
│  └── 通報対応                                           │
│                                                         │
│  一般ユーザー (USER)                                     │
│  ├── 商品登録（承認不要）                                 │
│  ├── タグ作成（承認不要）                                 │
│  ├── タグ編集（承認不要）                                 │
│  ├── タグエイリアス提案（承認必要）                        │
│  ├── タグ翻訳提案（承認必要）                             │
│  ├── タグ含意提案（承認必要）                             │
│  ├── いいね、所有済み登録                                │
│  └── 通報                                               │
│                                                         │
│  未ログインユーザー                                       │
│  └── 閲覧・検索のみ                                      │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### 承認が必要な操作

| 操作 | 承認要否 | 理由 |
|------|----------|------|
| 商品登録 | 不要 | データ充実を優先 |
| タグ作成 | 不要 | データ充実を優先 |
| タグ編集（既存タグへの付与） | 不要 | 編集履歴で追跡可能 |
| タグエイリアス設定 | **要承認** | 検索結果に大きく影響 |
| タグ翻訳設定 | **要承認** | 翻訳品質の担保 |
| タグ含意設定 | **要承認** | 検索ロジックに影響 |
| タグ階層設定 | **要承認** | データ構造に影響 |

## 5. モデレーション方針

### 現在のフロー

```
ユーザー通報 → 管理者確認 → 対応
```

### 将来的な拡張（コミュニティモデレーション）

```
ユーザー通報 → コミュニティ投票 → モデレーター確認 → 対応
               ↓
         自動対応（閾値超え時）
```

**注意**: 将来的な拡張であり、現時点では管理者対応のみ

## 6. 収益化方針

### 現在のスタンス

- **非営利**での運営
- サービス維持のための最小限のコスト負担

### 将来の可能性（検討事項）

| 手段 | 検討状況 | 備考 |
|------|----------|------|
| BOOTHアフィリエイト | 検討可 | BOOTH商品へのリンクにアフィリエイトを付与 |
| 広告 | 検討可 | ユーザー体験を損なわない範囲で |
| プレミアム会員 | 検討可 | 追加機能の提供 |

**原則**: 収益化はサービスの質を損なわない範囲で検討

## 7. 開発判断基準

### 新機能の採用基準

機能追加を検討する際は、以下の質問に「はい」と答えられるか確認する：

1. **検索品質の向上に貢献するか？**
2. **データベースの品質向上に貢献するか？**
3. **ユーザーが商品を見つけやすくなるか？**
4. **シンプルさを維持できるか？**

### 却下すべき機能の特徴

- 検索とは関係のないソーシャル機能
- システムの複雑性を大幅に増加させる機能
- 少数のユーザーにしか恩恵がない機能
- メンテナンスコストが高すぎる機能

## 9. 用語定義

| 用語 | 定義 |
|------|------|
| タグ | 商品に付与されるメタデータ。検索のキー |
| タグ含意 | AタグがあればBタグも自動的に含まれる関係 |
| タグ階層 | タグの親子関係（例: パンツ → 衣服） |
| タグエイリアス | 同義語の正規化（例: avatar → vrchat_avatar） |
| レーティング | 商品の年齢制限区分（General/Sensitive/Questionable/Explicit） |
| セーフサーチ | レーティングによるフィルタリング機能 |
