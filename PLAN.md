# Plan: PolySeek プロモーション動画制作（Remotion）

## 概要

BOOTHで理想の商品を探すのが大変という課題に対して、コミュニティ主導のタグシステムで検索性を向上させるPolySeekのプロモーション動画をRemotionで制作する。

---

## 動画仕様

### 基本情報

| 項目 | 仕様 |
|------|------|
| 動画尺 | 30-60秒 |
| 解像度 | 1920x1080（横型・16:9） |
| フレームレート | 30fps |
| 配信先 | X（Twitter） |
| トーン | ストーリーテリング重視 |
| 視点 | 2人称（視聴者自身に語りかける） |

### 音声・BGM

| 項目 | 仕様 |
|------|------|
| ナレーション | VOICEVOX（合成音声） |
| キャラクター | 後で選定（ずんだもん等） |
| ナレーション文体 | **短文・インパクト重視**（「探してる？見つからない？」形式） |
| BGM | ローファイ〜テック系（推奨） |
| 効果音 | クリック音、検索音、成功音など |

### カラーパレット

- **基調**: PolySeekアプリのテーマカラーに準拠
- **アクセント**: 課題シーンは暗め、解決シーンは明るめに

---

## 動画構成（シーン別）

### Scene 1: 導入・課題提示（0:00-0:15）

**目的**: 視聴者の共感を得る

**演出**:
- **実際のBOOTH画面をDevToolsで編集**してスクリーンショット撮影
  - ブラウザのDevTools（F12）でHTMLを一時的に書き換え
  - 商品名・出品者名 → サンプルテキストに変更
  - 商品サムネイル → `<img src>` をいらすとやのURLに差し替え
  - ※画像編集ソフト不要、フォント・レイアウトが本物と同一
- 「こたつ」で検索 → 無関係な商品が大量に表示されるノイズ
- ノイズ商品（衣装、テクスチャ、ギミック）にフォーカス演出
- **問題提起トランジション（モーショングラフィックス風）**:
  - BOOTHスクリーンショットをパーティクル（グリッド分割）で上方向に飛び散らせる（1秒）
  - 黒背景に問題提起テキストを下からスライドイン（1.5秒）
  - トーン: モダン・テック系、スムーズなイージング

**ナレーション例**（短文・インパクト重視）:
> 「衣装探し。見つからない。」
> 「検索しても、ノイズだらけ。」
> 「...どうする？」

**テキスト表示**:
- 問題提起: 「求めていない商品ばかり表示されて困ったことはありませんか？」
- （オプション）「402万件の注文」「58億円の市場規模」などの統計データ

---

### Scene 2: 解決策の提示（0:15-0:25）

**目的**: PolySeekのコンセプトを伝える

**演出**:
- 暗い画面から明るく切り替わるトランジション
- PolySeekロゴ登場
- 「コミュニティ主導のタグシステム」をテキストで表示

**ナレーション例**:
> 「PolySeekは、ユーザーコミュニティが作るタグで商品を検索できるサービスです」
> 「公式タグにはない『フリル』『サイバーパンク』といった詳細な特徴で検索できます」

---

### Scene 3: 機能デモ（0:25-0:50）

**目的**: 実際の使用感を見せる

#### 3-1: タグ検索デモ（0:25-0:35）
**演出**:
- 実際のPolySeek UI画面キャプチャ
- タグをクリックして絞り込む様子
- 検索結果がスムーズに表示される

**ナレーション例**:
> 「例えば『青いリボン』『レース生地』といったタグで検索すると...」
> 「一発で理想の商品が見つかります」

#### 3-2: タグ付け機能デモ（0:35-0:45）
**演出**:
- ユーザーがタグを追加する様子
- タグ候補のサジェスト表示
- タグが追加されるアニメーション

**ナレーション例**:
> 「見つけた商品には、あなた自身がタグを付けることができます」
> 「みんなでタグを付けることで、データベースがどんどん便利になっていきます」

#### 3-3: UI紹介（0:45-0:50）
**演出**:
- トップページ、検索結果ページ、商品詳細ページのハイライト
- モダンなUIデザインを強調

---

### Scene 4: CTA（0:50-0:60）

**目的**: 視聴者にアクションを促す

**演出**:
- PolySeekロゴ中央配置
- サービスURLを大きく表示
- 「今すぐ試す」などのアクションテキスト

**ナレーション例**:
> 「PolySeekで、理想のアセットを見つけよう」

**表示要素**:
- PolySeekロゴ
- サービスURL（polyseek.xxx）
- QRコード（オプション）

## 技術仕様（Remotion）

### プロジェクト構成

```
remotion/
├── src/
│   ├── Root.tsx           # エントリーポイント
│   ├── Video.tsx          # メインコンポジション
│   ├── scenes/
│   │   ├── Scene1Problem.tsx    # 課題提示シーン
│   │   ├── Scene2Solution.tsx   # 解決策シーン
│   │   ├── Scene3Demo.tsx       # デモシーン
│   │   └── Scene4CTA.tsx        # CTAシーン
│   ├── components/
│   │   ├── Logo.tsx
│   │   ├── TagAnimation.tsx
│   │   ├── SearchBar.tsx
│   │   └── ProductCard.tsx
│   ├── assets/
│   │   ├── images/        # スクリーンショット
│   │   ├── audio/         # BGM、SE、ナレーション
│   │   └── fonts/
│   └── styles/
│       └── global.css
├── package.json
└── remotion.config.ts
```

## 制作フロー

1. **素材準備**
   - UI画面のスクリーンショット撮影
   - 操作画面の録画
   - ナレーション原稿確定・音声生成
   - BGM・SE選定

2. **Remotionプロジェクトセットアップ**
   - プロジェクト初期化
   - コンポーネント設計

3. **シーン制作**
   - Scene1: 課題提示
   - Scene2: 解決策
   - Scene3: デモ
   - Scene4: CTA

4. **音声・BGM統合**
   - タイミング調整
   - 音量バランス調整

5. **レンダリング・確認**
   - プレビュー確認
   - 最終レンダリング

---

## 次のステップ

1. **素材収集**: PolySeekの現在のUI画面をキャプチャ
2. **ナレーション原稿の詳細化**: 各シーンのセリフを確定
3. **Remotionプロジェクトの初期化**: 技術環境のセットアップ
4. **VOICEVOXキャラクター選定**: ナレーション音声のトーン決定
5. **BGM選定**: 動画の雰囲気に合う楽曲を選定
